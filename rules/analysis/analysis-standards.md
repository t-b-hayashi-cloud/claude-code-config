# 分析作業標準ルール

## 1. ディレクトリ構成・命名規則

- ノートブック・スクリプト・出力ファイルには番号プレフィックスを付与し、対応関係を明示する
  - 形式: `{番号}-{説明的な名前}.{拡張子}`（例: `1.0-recency-analysis.ipynb`）
  - 単語区切りはハイフン
- 出力は必ず `outputs/` 配下に格納する
  - 図表: `outputs/figures/`
  - レポート: `outputs/reports/`

## 2. データ取り扱い

- 生データ（`data/raw/`）は読み取り専用とし、直接変更しない
- 中間データは `data/interim/`、最終データは `data/processed/` に保存する
- `data/` 内は Git 管理対象外とする
